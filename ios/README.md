# iOS 目录说明

当前 iOS 目录的目标不是只做页面壳子，而是逐步做成一个**纯 App 就能工作的单机 Demo**。

## 当前已有能力
- SwiftUI App 工程骨架
- 首页 / 龙虾 / 任务 / 审批 / 设置
- Mock 状态联动
- 审批、任务控制、配对等交互反馈
- 演示场景切换（标准 / 空状态 / 错误恢复）
- 本地状态持久化与恢复（基于 UserDefaults + Codable）

## 当前阶段原则
1. 先让纯 App 自己工作
2. 再把 Connector 当作后续接入层
3. 避免过早引入重型依赖或复杂持久化方案

## 后续方向
- 继续打磨纯 App 单机体验
- 再逐步把数据源从 Mock/本地状态切到 Connector
