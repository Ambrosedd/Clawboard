# 龙虾 OS 需求文档（App First 版）

## 1. 产品定位

### 1.1 一句话定义
一个 **App First、本地优先、隐私友好** 的龙虾管理应用。用户通过 iPhone App 直接连接自己电脑或服务器上的 Connector，查看龙虾状态、任务进度，并进行审批与控制。

### 1.2 当前阶段目标
当前阶段不做重后端，不做复杂云平台，优先验证：
- 用户是否愿意用手机管理龙虾
- 用户是否需要随时查看任务进度
- 用户是否需要在移动端完成审批和紧急接管

### 1.3 产品原则
1. **App First**：用户主要使用 iPhone App
2. **本地优先**：优先直连用户自己的机器
3. **低操作复杂度**：安装 Connector，扫码配对即可使用
4. **轻服务端预留**：未来可升级云同步、推送、协作
5. **默认安全克制**：不让中心服务直接拿机器高危权限

---

## 2. 用户故事

### 2.1 作为用户，我希望
- 在手机上看到所有龙虾当前状态
- 知道哪个龙虾正在工作、哪个卡住了
- 在任务异常时收到提醒并快速处理
- 在需要时批准某个敏感动作
- 直接在手机上暂停、恢复、终止一个龙虾

### 2.2 作为早期用户，我不希望
- 先搭一堆云服务才可使用
- 把我的服务器高危权限交给第三方后端
- 用复杂后台做大量配置
- 学太多企业级权限系统概念

## 3. MVP 范围

## 3.1 必做功能

### A. 设备与连接
- 添加 Connector 节点
- 扫码配对
- 查看设备在线状态
- 支持局域网 / 远程直连（由 Connector 决定）

### B. 龙虾列表
- 展示龙虾名称
- 当前状态：空闲 / 运行中 / 异常 / 等审批 / 暂停
- 当前任务摘要
- 最近活跃时间

### C. 龙虾详情
- 当前任务
- 当前步骤
- 最近日志摘要
- 当前建议动作
- 快捷控制：暂停 / 恢复 / 终止

### D. 任务详情
- 时间线
- 当前卡点
- 审批状态
- 输入摘要 / 输出摘要
- 错误原因

### E. 审批与告警
- 查看待审批项
- 批准 / 拒绝
- 查看异常提醒
- 对异常任务进行处理

### F. 本地连接管理
- 当前连接节点列表
- 节点名称
- 节点在线状态
- 最后同步时间

## 5. 关键流程

### 5.1 首次配对
1. 用户在电脑/服务器上启动 Connector
2. Connector 展示二维码或配对码
3. App 扫码、粘贴连接串，或自动读取剪贴板中的连接串
4. App 自动拆出配对信息并保存节点信息
5. App 开始拉取龙虾状态

### 5.2 查看任务进度
1. 用户打开 App 首页
2. 点击运行中的任务
3. 进入任务详情
4. 查看时间线、卡点、输出摘要

### 5.3 处理审批
1. 龙虾运行中触发敏感动作
2. Connector 上报待审批事件
3. App 审批页显示待审批项
4. 用户批准 / 拒绝
5. 结果回传给 Connector

### 5.4 紧急干预
1. 龙虾任务异常
2. App 展示异常提醒
3. 用户进入龙虾详情或任务详情
4. 执行暂停 / 恢复 / 终止

---

## 6. 界面设计原则
1. 简洁优先，不做复杂后台感
2. 摘要优先，详情下钻
3. 高风险状态清晰可见
4. 交互动作少而明确
5. 优先适配 iPhone 单手操作

## 8. 当前结论
当前产品不是一个企业级管理后台，而是一个：

> **通过 iPhone 管理龙虾工作的本地优先控制 App**

优先验证移动端管理场景，再决定是否演进到轻量服务端。
