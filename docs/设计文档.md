# 龙虾 OS 设计文档（App First 版）

## 1. 总体思路

当前版本采用：

```text
iPhone App (SwiftUI)
        ↓
   Direct Connection
        ↓
Connector（运行在用户电脑/服务器）
        ↓
Lobster Runtime / Agent Runtime
```

目标：
- 不依赖重后端
- 用户主要使用 App
- 电脑/服务器上的 Connector 负责本地执行与状态暴露
- 未来再演进为可选轻量服务端

---

## 2. 角色分工

## 2.1 iPhone App
职责：
- 展示龙虾列表与状态
- 展示任务列表与详情
- 展示审批与告警
- 发起控制命令（暂停/恢复/终止/批准/拒绝）
- 管理节点连接与配对
- 本地缓存最近状态

不负责：
- 长时间后台常驻执行
- 保存高危机器权限
- 执行龙虾工作流

---

## 2.2 Connector
运行在用户电脑或服务器上。

职责：
- 暴露一个受控接口给 App
- 获取本机龙虾运行状态
- 读取任务事件流
- 接收 App 控制命令
- 执行白名单动作
- 本地持有必要权限和凭证

不负责：
- 复杂产品 UI
- 多租户后台
- 云端团队协作

---

## 2.3 Lobster Runtime
职责：
- 执行具体 agent 任务
- 输出任务状态、步骤、日志、审批事件
- 接收暂停/恢复/终止等控制命令

可选实现：
- Software-team
- 自研脚本
- 其他 agent runtime

---

## 3. 架构原则
1. **App 是主控制台**
2. **Connector 是本地执行边界**
3. **高危凭证只留在本地**
4. **接口模型稳定，方便未来接云中继**
5. **先单机/少量节点，再演进多节点**

---

## 4. 连接模式

## 4.1 当前推荐模式
- App 通过 HTTPS / WebSocket 直接连接 Connector
- 配对方式：二维码 / 配对码
- Connector 可运行在局域网、内网穿透网络或用户自定义远程地址上

## 4.2 未来升级模式
未来可以增加：
- Relay / Cloud Connector
- APNs 推送中转
- 多端同步

但这些不属于当前 MVP。

## 5. 接口设计（App ↔ Connector）

## 5.1 基础接口
- `GET /health`
- `GET /device/info`
- `GET /lobsters`
- `GET /lobsters/:id`
- `GET /tasks`
- `GET /tasks/:id`
- `GET /approvals`
- `GET /alerts`

## 5.2 控制接口
- `POST /lobsters/:id/pause`
- `POST /lobsters/:id/resume`
- `POST /lobsters/:id/terminate`
- `POST /approvals/:id/approve`
- `POST /approvals/:id/reject`
- `POST /tasks/:id/retry`

## 5.3 实时接口
- `GET /stream/events`（SSE 或 WebSocket）

事件包括：
- lobster.status.changed
- task.progress.updated
- task.waiting_approval
- task.failed
- approval.created
- alert.created

## 6. Connector 内部模块

## 6.1 Runtime Adapter
将不同 agent runtime 映射为统一事件模型。

## 6.2 State Cache
缓存龙虾列表、任务摘要、审批与告警，方便 App 拉取。

## 6.3 Command Executor
接收 App 指令并执行有限动作。

## 6.4 Security Guard
负责：
- 配对校验
- token 校验
- 动作白名单
- 本地权限边界

## 9. 安全方案

## 9.1 当前阶段安全原则
- 中心服务缺席或极弱存在
- 高危权限不离开用户机器
- Connector 仅暴露必要接口
- 所有控制动作要求 token 校验
- 支持手动撤销配对

## 9.2 配对机制
推荐：
- Connector 首次启动生成一次性配对码
- App 扫码后换取本地长期 token
- token 存储在 iOS Keychain

## 9.3 命令约束
Connector 仅允许：
- 查看状态
- 查看任务
- 批准/拒绝
- 暂停/恢复/终止

不允许默认开放：
- 任意 shell 执行
- 任意文件读取
- 任意数据库查询

## 11. 开发阶段建议

## Phase 1：纯 App MVP
- SwiftUI App
- 单 Connector 配对
- 龙虾列表
- 任务详情
- 审批页
- 基础控制动作

## Phase 2：体验增强
- 实时事件流
- 本地通知
- 多 Connector 管理
- 更好的缓存与离线展示

## Phase 3：可选云增强
- Relay
- Push 中继
- 多设备同步
- 云端事件历史
- 团队协作

---

## 12. 当前技术判断
当前不适合优先做：
- 重后端
- 企业权限平台
- 复杂管理后台

当前最适合优先做：
- 一个美观、简单、可直连 Connector 的 iPhone App
- 一个受控、轻量、安全边界清晰的 Connector

---

## 13. 总结
当前版本本质上是：

> **一个用 iPhone 直接管理龙虾运行端的 App First 产品。**

技术上先走最短路径，把“观察 + 审批 + 控制”做对，再考虑服务升级与商业化。
