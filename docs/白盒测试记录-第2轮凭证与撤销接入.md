# 白盒测试记录：第 2 轮凭证与撤销接入

## 范围
本轮主要覆盖：
- Bridge token 会话查看接口
- Bridge token 撤销接口
- iOS 侧凭证存储边界抽象
- iOS 侧断开连接时 revoke 调用逻辑的静态检查

## 测试方法
### 1. 本地启动 Bridge
使用 Node 启动 `connector/src/server.js`。

### 2. 配对换 token
调用：
- `POST /pair/exchange`

### 3. 查看 token 当前会话
调用：
- `GET /auth/session`

确认返回：
- `node`
- `session.token_preview`
- `session.created_at`
- `session.client`

### 4. 撤销 token
调用：
- `POST /auth/revoke`

### 5. 撤销后再次访问受保护接口
调用：
- `GET /tasks`

确认撤销后的 token 不再可用。

## 实际结果
### `/auth/session`
通过，能返回当前 token 对应的会话摘要。

### `/auth/revoke`
通过，返回：
- `ok: true`
- `revoked_at: ...`

### 撤销后访问 `/tasks`
通过，返回：
- `401 unauthorized`
- `missing or invalid bearer token`

## iOS 静态审查结果
### 已完成
- Bridge 凭证已从普通 `PersistedAppState` 中拆出
- 引入 `BridgeCredentialStore` 抽象
- 当前先用 UserDefaults 模拟 Keychain 边界
- `disconnectBridge()` 已尝试调用 revoke，再清理本地凭证

### 仍未完成
- 真正的 iOS Keychain 实现
- token 刷新/续签策略
- 多设备 token 管理界面
- 撤销失败时的 UI 级可见反馈

## 结论
本轮已经把 token 生命周期从“只会签发”推进到了：
- 可查看当前会话摘要
- 可主动撤销
- 撤销后立即失效

这让后续的“断开连接 / 重新配对 / 多设备管理”有了可继续扩展的基础。
