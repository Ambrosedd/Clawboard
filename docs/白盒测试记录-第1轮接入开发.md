# 白盒测试记录：第 1 轮接入开发

## 范围
本轮白盒测试主要覆盖：
- Bridge 配对协议骨架
- Pair session / exchange 接口
- Bearer Token 鉴权路径的基础建立
- iOS 连接建模的静态审查
- skill 目录结构、frontmatter 与内容完整性

## 测试方法
### 1. Bridge API 直接调用
使用本地 Node 进程启动 `connector/src/server.js`，再用 `curl` 调用：
- `GET /pair/session`
- `POST /pair/exchange`

### 2. 返回值白盒检查
确认：
- pair session 返回 `pairing_id / pair_code / expires_at / node_id`
- exchange 返回 `token / token_type / issued_at / node`
- 生成 token 符合 `cb_live_*` 前缀约定

### 3. iOS 代码静态检查
由于当前环境无 `swift` / Xcode，不能做编译级验证，因此改用：
- 结构级检查
- 模型映射检查
- 逻辑路径检查
- 明显语义错误排查

### 4. skill 结构检查
检查：
- `skills/clawboard-bridge/SKILL.md`
- `references/pairing.md`
- `references/authorization.md`
- `references/lifecycle.md`
- 使用 `quick_validate.py` 运行结构校验

## 当前已知限制
1. 当前环境没有 `swift` 命令，无法进行 Swift 编译级白盒测试。
2. iOS 连接逻辑目前仍未做真机 Keychain 存储。
3. 真实 Bridge 控制命令尚未回写到远端真实 runtime，仅完成骨架形状。
4. 真实 Lobster skill 还停留在设计和目录级交付，未接入实际 Lobster runtime 生命周期。
5. 当前 Bearer Token 仅做内存态签发，服务重启后不会保留。

## 当前结论
本轮已经完成：
- Pair 协议骨架
- Bridge 侧 pair API 骨架
- iOS 侧 bridge connection 建模与真实配对请求路径
- skill 设计文档与目录结构
