# 决策日志-第8轮事件流骨架

## 本轮目标
把 Clawboard Bridge 从“纯拉取型骨架”继续推进到“具备实时事件流能力的骨架”。

---

## 1. 为什么这轮先补 SSE

上一轮已经把 App 侧的真实控制闭环补上了，但仍然有明显短板：
- 操作成功后主要靠 refresh 重新拉取
- Bridge 没有标准化的实时事件出口
- 后续如果要做推送式 UI 更新、通知、或 skill-hosted sidecar 对 runtime 的桥接，都缺一个稳定事件层

所以这轮优先补的是：

> **先把服务端的事件契约稳定下来。**

这比立即在 iOS 端做复杂长连接更划算，因为：
1. 事件语义一旦稳定，多个客户端都能复用
2. runtime adapter 接进来时，也有明确的事件出口
3. 即使 App 暂时仍以 refresh 为主，服务端也已经不是“只能轮询”的形态

---

## 2. 为什么选 SSE 而不是先上 WebSocket

### 方案 A：直接 WebSocket
**优点**
- 双向通信能力更强
- 后续扩展性高

**缺点**
- 对当前 MVP 过重
- 服务端实现和 App 端维护成本更高
- 现在主要需求是“服务端向 App 推送事件”，不是复杂双向信道

### 方案 B：先上 SSE
**优点**
- 足够覆盖当前“单向事件推送”场景
- 基于 HTTP，骨架实现更轻
- 容易调试，事件格式更直接
- 后续如果真要升级 WebSocket，也不会浪费已有事件模型

**缺点**
- 双向能力弱
- 重连和状态恢复要自己约定 `Last-Event-ID`

### 当前选择
选择 **SSE**。

这更适合当前阶段：
> 先把实时“看见变化”这件事做对，再考虑是否需要更重的实时双向通道。

## 4. 一个中途发现并修正的问题

在第一轮 smoke test 时发现：
- `/health` 也被统一鉴权拦住了

这不合理。

### 为什么不合理
`/health` 的定位是：
- 判断 bridge 是否存活
- 支持部署、诊断、探测
- 不应要求先完成配对和持有 token

如果 `/health` 都被鉴权挡住，会让：
- 首次连接诊断更麻烦
- 用户填地址后无法先做最基础探活
- 运维与测试体验变差

### 当前修正
将 `/health` 改为免鉴权。

### 仍保持鉴权的
`/stream/events` 继续要求 Bearer token。

原因是事件流里包含：
- 节点变化
- 审批处理结果
- 任务状态变化

这已经超出纯探活信息，应该保留授权边界。

## 5. 验证结果

已做本地 smoke test，验证通过：
1. 启动 bridge
2. 获取 pair code
3. exchange 换 token
4. 带 token 建立 `/stream/events`
5. 执行 approval / pause / retry
6. 成功在 SSE 中收到连续事件

实际观察到的代表性事件序列包括：
- `bridge.started`
- `pair.exchanged`
- `approval.resolved`
- `task.progress.updated`
- `lobster.status.changed`

说明当前 SSE 骨架已经不是文档占位，而是可工作的接口。

---

## 6. 这一轮的价值

这轮完成后，Clawboard Bridge 已经从：
- 只能轮询拉取状态

推进到：
- **具备实时事件输出能力的本地 bridge 骨架**

这对后续几件事都非常关键：
1. iOS App 接增量更新
2. 未来本地通知/告警桥接
3. runtime adapter 接入时输出统一事件
4. sidecar 生命周期与诊断能力增强

---

## 7. 下一步最值得继续补什么

按性价比排序，下一阶段更值得补的是：

### A. iOS 端消费 SSE
让 App 在真实 Bridge 连接时，逐步摆脱“操作后 refresh”的体验。

### B. 更真实的 runtime adapter
把 bridge 内部状态源从种子数据继续往“真实服务器上的 lobster 运行态”推进。

### C. token / session 管理 UI
把 `auth/session` 与 `auth/revoke` 进一步带进 App，形成可见可控的连接管理。

---

## 8. 当前结论

这一轮不是“看起来更完整”的改动，而是：

> **把 Bridge 正式推进到具备实时事件能力的产品骨架。**

这一步让后面的 App 实时化、通知化、以及 runtime 深接入，都有了稳定支点。
