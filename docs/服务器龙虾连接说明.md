# 服务器龙虾连接说明

这份说明只回答一个问题：

> **用户怎样把服务器上的龙虾快速连到 Clawboard App？**

当前阶段的目标不是暴露技术细节，而是先把连接路径收顺。

---

## 1. 用户应该看到什么

用户不应该先理解这些词：
- Connector
- Sidecar
- Bridge
- Runtime Adapter

用户应该看到的是：
- 安装 Clawboard skill
- 启用移动连接
- 获得配对码 / 二维码
- 在 App 中"添加龙虾"

---

## 2. 当前推荐连接路径

### 服务器侧
1. 安装 `clawboard-bridge` skill
2. 启用 skill
3. skill 拉起本地 bridge
4. bridge 显示：
   - 节点名
   - 配对码
   - 二维码（后续补）
   - 过期时间

### App 侧
1. 打开 Clawboard App
2. 进入“设置” → “添加龙虾”
3. 复制 App 提供的下载链接或一键安装命令
4. 把这段命令发给龙虾执行，让龙虾安装 skill
5. 再让龙虾运行 `start-bridge.sh` 与 `show-connection.sh`
6. 把返回的连接串粘贴回 App 完成连接

仅在调试/自建环境异常时，才进入“高级连接选项”手动填写 Bridge 地址或配对码。

---

## 3. 为什么把地址放进高级选项，并引入连接串

### 备选方案
1. 直接让用户先填地址 + 配对码
2. 先让用户只输入配对码，地址折叠进高级项

### 对比
#### 方案 1 优点
- 更贴近当前底层实现
- 调试方便

#### 方案 1 缺点
- 普通用户不知道什么是 Bridge 地址
- 破坏"添加龙虾"的产品心智

#### 方案 2 优点
- 用户只需要关注"配对码"
- 地址只在调试/自建环境时暴露
- 更接近未来扫码连接形态

#### 方案 2 缺点
- 默认地址策略需要更明确

### 当前选择
选择 **方案 2**。

---

## 4. 前期未进官方库时，怎么安装 skill

### 当前建议
前期用这套组合：
1. **可交付 skill bundle**（skill + bridge runtime + 启停脚本）
2. **一键安装脚本**（降低操作门槛）
3. **极短安装说明**（避免把用户拖进工程细节）

### 不推荐只给什么
- 不推荐只给源码仓库
- 不推荐只给提示词
- 不推荐要求用户手动理解 skill 内部结构

---

## 5. 后续要继续补什么

1. 扫码连接（替代手输配对码）
2. 安装完成后自动显示二维码
3. 更明确的 `lobster mobile enable` / `lobster pair` 入口
4. Keychain 存储与 token 撤销管理
5. 技术文档和用户说明分层

---

## 6. 当前一句话结论

**先把"添加龙虾"做成用户能理解的路径，再把 skill 官方库接入作为后续分发优化。**
